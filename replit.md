# WhatsApp AI Bot

A WhatsApp bot server that receives messages via Twilio's WhatsApp sandbox and replies using Claude (Anthropic) AI.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 5000)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- WhatsApp: Twilio WhatsApp Sandbox
- AI: Anthropic Claude (claude-3-5-haiku)

## Where things live

- `artifacts/api-server/src/routes/whatsapp.ts` — webhook route handler (GET + POST)
- `artifacts/api-server/src/lib/claude.ts` — Anthropic Claude integration
- `artifacts/api-server/src/lib/twilio.ts` — Twilio client + signature validation

## Webhook Endpoints

- `GET /api/whatsapp/webhook` — health check, returns webhook URL
- `POST /api/whatsapp/webhook` — receives incoming WhatsApp messages from Twilio, replies via Claude

## Architecture decisions

- Twilio signature validation is performed on every POST; if the header is missing (dev/testing), validation is skipped with a warning log.
- Claude uses `claude-3-5-haiku` for fast, concise chat-appropriate replies.
- Replies are sent back via Twilio REST API (not TwiML response body) for reliability.

## Product

Users can message the WhatsApp sandbox number and receive AI-powered replies generated by Claude. The bot keeps responses concise and plain-text friendly for the WhatsApp chat format.

## User preferences

- Use Twilio WhatsApp Sandbox (not Meta Graph API directly)
- Use Anthropic Claude for AI responses

## Gotchas

- The webhook URL shown by GET /api/whatsapp/webhook uses the internal `localhost` host. When registering in Twilio, use the public Replit domain: `https://<your-repl>.replit.app/api/whatsapp/webhook`
- Twilio sends form-encoded POST bodies; `express.urlencoded` middleware is required (already configured in app.ts).
